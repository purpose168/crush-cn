# Crush-CN 配置说明

## 配置文件位置

Crush-CN 的配置文件位于以下位置：

| 操作系统 | 配置目录 |
|----------|----------|
| Linux/macOS | `~/.config/crush/` |
| Windows | `%LOCALAPPDATA%\crush\` |
| Android (Termux) | `~/.config/crush/` |

### 主要配置文件

| 文件 | 说明 |
|------|------|
| `config.json` | 主配置文件 |
| `providers.json` | LLM 提供商配置 |
| `mcp.json` | MCP 服务器配置 |
| `skills/` | 技能目录 |

## 主配置文件 (config.json)

### 配置结构

```json
{
  "model": "gpt-4",
  "provider": "openai",
  "session": {
    "auto_save": true,
    "max_history": 100
  },
  "ui": {
    "theme": "dark",
    "show_token_count": true
  },
  "output": {
    "format": "markdown",
    "code_highlight": true
  }
}
```

### 配置项说明

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `model` | string | 默认使用的模型 | `gpt-4` |
| `provider` | string | 默认提供商 | `openai` |
| `session.auto_save` | bool | 自动保存会话 | `true` |
| `session.max_history` | int | 最大历史记录数 | `100` |
| `ui.theme` | string | UI 主题 | `dark` |
| `ui.show_token_count` | bool | 显示 token 计数 | `true` |
| `output.format` | string | 输出格式 | `markdown` |
| `output.code_highlight` | bool | 代码高亮 | `true` |

### 配置结构

```json
{
  "providers": [
    {
      "id": "openai",
      "name": "OpenAI",
      "base_url": "https://api.openai.com/v1",
      "type": "openai",
      "api_key": "sk-xxx...",
      "models": [
        {
          "id": "gpt-4",
          "name": "GPT-4",
          "context_window": 128000,
          "max_output_tokens": 4096
        },
        {
          "id": "gpt-4-turbo",
          "name": "GPT-4 Turbo",
          "context_window": 128000,
          "max_output_tokens": 4096
        },
        {
          "id": "gpt-3.5-turbo",
          "name": "GPT-3.5 Turbo",
          "context_window": 16385,
          "max_output_tokens": 4096
        }
      ]
    },
    {
      "id": "anthropic",
      "name": "Anthropic",
      "base_url": "https://api.anthropic.com/v1",
      "type": "anthropic",
      "api_key": "sk-ant-xxx...",
      "models": [
        {
          "id": "claude-3-opus",
          "name": "Claude 3 Opus",
          "context_window": 200000,
          "max_output_tokens": 4096
        },
        {
          "id": "claude-3-sonnet",
          "name": "Claude 3 Sonnet",
          "context_window": 200000,
          "max_output_tokens": 4096
        }
      ]
    }
  ]
}
```

### 提供商配置项

| 配置项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| `id` | string | 是 | 提供商唯一标识 |
| `name` | string | 是 | 提供商显示名称 |
| `base_url` | string | 是 | API 基础 URL |
| `type` | string | 是 | 提供商类型（openai/anthropic） |
| `api_key` | string | 是 | API 密钥 |
| `models` | array | 是 | 可用模型列表 |

### 模型配置项

| 配置项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| `id` | string | 是 | 模型唯一标识 |
| `name` | string | 是 | 模型显示名称 |
| `context_window` | int | 否 | 上下文窗口大小 |
| `max_output_tokens` | int | 否 | 最大输出 token 数 |

### 支持的提供商类型

| 类型 | 说明 | API 格式 |
|------|------|----------|
| `openai` | OpenAI 及兼容 API | OpenAI API |
| `anthropic` | Anthropic Claude | Anthropic API |

## MCP 服务器配置 (mcp.json)

### 配置结构

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "mcp-filesystem",
      "args": ["/path/to/allowed/directory"],
      "env": {}
    },
    "github": {
      "command": "mcp-github",
      "args": [],
      "env": {
        "GITHUB_TOKEN": "ghp_xxx..."
      }
    },
    "http-server": {
      "url": "http://localhost:8080/mcp",
      "type": "http"
    },
    "sse-server": {
      "url": "http://localhost:8081/sse",
      "type": "sse"
    }
  }
}
```

### MCP 服务器类型

| 类型 | 说明 | 配置方式 |
|------|------|----------|
| `stdio` | 标准输入输出 | 使用 `command` 和 `args` |
| `http` | HTTP 连接 | 使用 `url` 和 `type: http` |
| `sse` | Server-Sent Events | 使用 `url` 和 `type: sse` |

### stdio 类型配置

```json
{
  "command": "mcp-server-name",
  "args": ["--option", "value"],
  "env": {
    "ENV_VAR": "value"
  }
}
```

### HTTP/SSE 类型配置

```json
{
  "url": "http://localhost:8080/mcp",
  "type": "http",
  "headers": {
    "Authorization": "Bearer token"
  }
}
```

```markdown
你是一个专业的代码审查助手。请审查用户提供的代码，并提供以下方面的反馈：

1. 代码质量

```

## 环境变量配置

### API 密钥环境变量

| 变量 | 说明 |
|------|------|
| `OPENAI_API_KEY` | OpenAI API 密钥 |
| `ANTHROPIC_API_KEY` | Anthropic API 密钥 |
| `GITHUB_TOKEN` | GitHub 个人访问令牌 |

### 其他环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CRUSH_CONFIG_PATH` | 配置文件路径 | `~/.config/crush` |
| `CRUSH_DATA_PATH` | 数据文件路径 | `~/.local/share/crush` |
| `CRUSH_CACHE_PATH` | 缓存文件路径 | `~/.cache/crush` |
| `CRUSH_PROFILE` | 启用性能分析 | `false` |
| `NO_COLOR` | 禁用颜色输出 | `false` |

## 命令行配置

### 查看配置

```bash
# 查看当前配置
crush config

# 查看特定配置项
crush config get model
crush config get provider

# 查看所有提供商
crush providers
```

### 设置配置

```bash
# 设置默认模型
crush config set model gpt-4-turbo

# 设置默认提供商
crush config set provider openai

# 设置 API 密钥
crush config set api_key sk-xxx...
```

### 管理会话

```bash
# 列出所有会话
crush sessions

# 创建新会话
crush new

# 切换会话
crush switch <session-id>

# 删除会话
crush delete <session-id>
```

## 配置最佳实践

### 1. API 密钥安全

- 不要将 API 密钥硬编码在配置文件中
- 使用环境变量存储敏感信息
- 使用 `${ENV_VAR}` 语法引用环境变量

```json
{
  "api_key": "${OPENAI_API_KEY}"
}
```

### 2. 配置文件备份

```bash
# 备份配置
cp -r ~/.config/crush ~/.config/crush.backup

# 或使用版本控制
cd ~/.config/crush
git init
git add .
git commit -m "备份配置"
```

### 3. 多环境配置

```bash
# 使用环境变量切换配置
export CRUSH_CONFIG_PATH=~/.config/crush-work

# 或创建符号链接
ln -s ~/.config/crush-personal ~/.config/crush
```

### 4. 模型选择建议

| 场景 | 推荐模型 |
|------|----------|
| 代码生成 | GPT-4, Claude 3 Opus |
| 快速响应 | GPT-3.5 Turbo, Claude 3 Sonnet |
| 长文本处理 | Claude 3 (200K context) |
| 本地部署 | Llama 2, Code Llama |

## 故障排除

### 1. 配置文件未找到

**问题**：启动时提示配置文件不存在。

**解决方案**：
```bash
# 创建配置目录
mkdir -p ~/.config/crush

# 生成默认配置
crush config init
```

### 2. API 密钥无效

**问题**：API 调用失败，提示认证错误。

**解决方案**：
```bash
# 检查 API 密钥
crush config get api_key

# 更新 API 密钥
crush config set api_key sk-xxx...

# 或使用环境变量
export OPENAI_API_KEY=sk-xxx...
```

### 3. 模型不可用

**问题**：选择的模型不在可用列表中。

**解决方案**：
```bash
# 查看可用模型
crush models

# 更新提供商配置
# 编辑 ~/.config/crush/providers.json
```

### 4. MCP 服务器连接失败

**问题**：MCP 服务器无法连接。

**解决方案**：
```bash
# 检查 MCP 配置
cat ~/.config/crush/mcp.json

# 测试 MCP 服务器
mcp-filesystem /path/to/directory

# 查看日志
crush --debug
```
