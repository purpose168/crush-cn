## 快速导航

### 按角色导航

#### 新加入开发者
建议阅读顺序：
1. 项目架构概述 → 了解系统整体设计
2. 技术栈说明 → 熟悉使用的技术
3. 模块划分 → 理解模块职责
4. 开发规范 → 掌握编码标准
5. 构建部署流程 → 学习开发环境搭建

#### 架构师
建议阅读顺序：
1. 项目架构概述 → 理解架构设计
2. 模块划分 → 分析模块边界
3. 接口定义 → 了解接口设计
4. 数据流程 → 理解数据流转
5. 版本控制策略 → 了解发布流程

#### 运维工程师
建议阅读顺序：
1. 技术栈说明 → 了解技术依赖
2. 构建部署流程 → 掌握部署方法
3. 测试策略 → 了解测试要求
4. 常见问题解决方案 → 掌握故障处理

#### 测试工程师
建议阅读顺序：
1. 项目架构概述 → 了解系统结构
2. 模块划分 → 理解功能模块
3. 接口定义 → 了解测试接口
4. 测试策略 → 掌握测试方法

### 按任务导航

#### 添加新功能
1. 查阅 [模块划分](./03-模块划分.md) 确定功能归属
2. 参考 [接口定义](./04-接口定义.md) 设计接口
3. 遵循 [开发规范](./06-开发规范.md) 编写代码
4. 按照 [测试策略](./08-测试策略.md) 编写测试

#### 修复 Bug
1. 参考 [常见问题解决方案](./10-常见问题解决方案.md) 排查问题
2. 查阅 [数据流程](./05-数据流程.md) 理解数据流转
3. 遵循 [开发规范](./06-开发规范.md) 修复问题
4. 按照 [测试策略](./08-测试策略.md) 验证修复

#### 部署上线
1. 参考 [构建部署流程](./07-构建部署流程.md) 准备部署
2. 查阅 [版本控制策略](./09-版本控制策略.md) 确定版本
3. 按照 [测试策略](./08-测试策略.md) 执行测试
4. 参考 [常见问题解决方案](./10-常见问题解决方案.md) 处理问题
