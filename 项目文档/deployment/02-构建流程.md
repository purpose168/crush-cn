# Crush-CN 构建流程

## 构建概述

Crush-CN 使用 Taskfile 作为任务运行器，提供了完整的构建、测试、检查和发布流程。本文档详细说明项目的构建过程。

## Taskfile 任务

### 可用任务列表

| 任务 | 说明 | 命令 |
|------|------|------|
| `lint:install` | 安装 golangci-lint | `task lint:install` |
| `lint` | 运行代码检查 | `task lint` |
| `lint:log` | 检查日志消息格式 | `task lint:log` |
| `lint:fix` | 运行代码检查并修复 | `task lint:fix` |
| `build` | 构建项目 | `task build` |
| `run` | 构建并运行 | `task run` |
| `test` | 运行测试 | `task test` |
| `test:record` | 运行测试并重新记录 VCR 磁带 | `task test:record` |
| `fmt` | 格式化代码 | `task fmt` |
| `fmt:html` | 格式化 HTML/CSS/JS 文件 | `task fmt:html` |
| `dev` | 启用性能分析运行 | `task dev` |
| `install` | 安装应用程序 | `task install` |
| `profile:cpu` | CPU 性能分析 | `task profile:cpu` |
| `profile:heap` | 堆内存性能分析 | `task profile:heap` |
| `profile:allocs` | 内存分配性能分析 | `task profile:allocs` |
| `schema` | 生成配置 JSON 架构 | `task schema` |
| `hyper` | 更新 Hyper provider.json | `task hyper` |
| `release` | 创建并推送新标签 | `task release` |
| `deps` | 更新依赖 | `task deps` |

## 构建流程

### 1. 获取源代码

```bash
# 克隆仓库
git clone https://github.com/purpose168/crush-cn.git
cd crush-cn

# 获取所有标签
git fetch --tags
```

### 2. 安装依赖

```bash
# 下载依赖
go mod download

# 整理依赖
go mod tidy
```

### 3. 代码格式化

```bash
# 使用 Taskfile
task fmt

# 或直接使用 gofumpt
gofumpt -w .
```

### 4. 代码检查

```bash

# 安装 golangci-lint
GOTOOLCHAIN=auto go install github.com/golangci/golangci-lint/v2/cmd/golangci-lint@latest


# 安装 golangci-lint
task lint:install

# 运行代码检查
task lint

# 运行代码检查并自动修复
task lint:fix
```

### 5. 运行测试

```bash
# 运行所有测试
task test

# 运行特定测试
task test -- -run TestAgent

# 运行测试并重新记录 VCR 磁带
task test:record
```

### 6. 构建项目

```bash
# 构建项目
task build

# 构建并运行
task run

# 带参数运行
task run -- --help
```

### 7. 安装到系统

```bash
# 安装应用程序
task install

# 验证安装
crush --version
```

## 详细任务说明

### lint:install

安装 golangci-lint 代码检查工具。

```bash
task lint:install
```

**环境要求**：
- Go 1.25.0+

### lint

运行代码检查，包括日志消息格式检查。

```bash
task lint
```

**检查内容**：
- 代码质量检查
- 日志消息大写检查
- 代码风格检查

### lint:fix

运行代码检查并自动修复可修复的问题。

```bash
task lint:fix
```

### build

构建项目，生成可执行文件。

```bash
task build
```

**生成的文件**：
- `crush`（Linux/macOS）
- `crush.exe`（Windows）

**构建标志**：
- `-ldflags`：设置版本信息
- `-race`：启用竞态检测（如果存在 race.log）

### run

构建并运行项目。

```bash
# 基本运行
task run

# 带参数运行
task run -- --help
task run -- --model gpt-4

# 启用竞态检测
touch race.log
task run
```

### test

运行测试，启用竞态检测。

```bash
# 运行所有测试
task test

# 运行特定测试
task test -- -run TestAgent

# 详细输出
task test -- -v

# 运行特定包的测试
task test -- ./internal/agent
```

### test:record

运行测试并重新记录所有 VCR 磁带（用于 API 测试录制）。

```bash
task test:record
```

**注意**：此任务会删除旧的测试数据并重新录制。

### fmt

格式化 Go 代码。

```bash
task fmt
```

使用 gofumpt 进行代码格式化。

### dev

启用性能分析运行项目。

```bash
task dev
```

**环境变量**：
- `CRUSH_PROFILE=true`

### install

安装应用程序到 `$GOPATH/bin`。

```bash
task install
```

### profile:cpu

进行 10 秒 CPU 性能分析。

```bash
# 首先启动应用
task dev &

# 在另一个终端运行
task profile:cpu
```

### profile:heap

进行堆内存性能分析。

```bash
task profile:heap
```

### profile:allocs

进行内存分配性能分析。

```bash
task profile:allocs
```

### schema

生成配置的 JSON 架构文件。

```bash
task schema
```

**生成的文件**：
- `schema.json`

### hyper

更新 Hyper 嵌入式 provider.json。

```bash
task hyper
```

### release

创建并推送遵循语义化版本的新标签。

```bash
task release
```

**前置条件**：
- 必须在 main 分支
- Git 工作区必须干净
- 构建测试必须通过
- 快照构建必须通过

### deps

更新 Fantasy 和 Catwalk 依赖。

```bash
task deps
```

## 环境变量

### 构建环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CGO_ENABLED` | 是否启用 CGO | `0` |
| `GOEXPERIMENT` | Go 实验性功能 | `greenteagc` |
| `VERSION` | 版本信息 | `git describe --long` |
| `RACE` | 是否启用竞态检测 | 检查 race.log 是否存在 |

### 性能分析环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CRUSH_PROFILE` | 启用性能分析 | `false` |

## 跨平台构建

### 使用 Go 直接构建

```bash
# Linux amd64
GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build -o crush-linux-amd64 .

# Linux arm64
GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -o crush-linux-arm64 .

# macOS amd64
GOOS=darwin GOARCH=amd64 CGO_ENABLED=0 go build -o crush-darwin-amd64 .

# macOS arm64 (Apple Silicon)
GOOS=darwin GOARCH=arm64 CGO_ENABLED=0 go build -o crush-darwin-arm64 .

# Windows amd64
GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build -o crush-windows-amd64.exe .

# FreeBSD amd64
GOOS=freebsd GOARCH=amd64 CGO_ENABLED=0 go build -o crush-freebsd-amd64 .
```

### 使用 GoReleaser 构建

```bash
# 安装 GoReleaser
go install github.com/goreleaser/goreleaser/v2@latest

# 构建快照版本（不发布）
goreleaser build --snapshot --clean

# 构建发布版本（需要 Git tag）
goreleaser release
```

## 构建优化

### 减小二进制文件大小

```bash
# 使用 -ldflags 去除调试信息
go build -ldflags="-s -w" -o crush .

# 使用 UPX 压缩
upx --best --lzma crush
```

### 构建缓存

```bash
# 查看构建缓存
go env GOCACHE

# 清理构建缓存
go clean -cache

# 清理测试缓存
go clean -testcache
```

## 常见构建问题

### 1. 版本信息未设置

**问题**：构建的二进制文件版本显示为空。

**解决方案**：
```bash
# 获取标签
git fetch --tags

# 使用 Taskfile 构建（自动设置版本）
task build

# 或手动设置
go build -ldflags="-X github.com/purpose168/crush-cn/internal/version.Version=$(git describe --long)" .
```

### 2. 依赖版本冲突

**问题**：
```
go: github.com/xxx@v1.0.0 requires github.com/yyy@v2.0.0
```

**解决方案**：
```bash
# 更新依赖
go get -u ./...
go mod tidy

# 或使用 Taskfile
task deps
```

### 3. 测试超时

**问题**：
```
panic: test timed out after 10m0s
```

**解决方案**：
```bash
# 增加超时时间
go test -timeout 30m ./...

# 或使用 Taskfile
task test:record  # 已设置 1 小时超时
```

### 4. 竞态检测问题

**问题**：
```
==================
WARNING: DATA RACE
==================
```

**解决方案**：
```bash
# 查看详细竞态信息
task test 2>race.log

# 分析竞态日志
cat race.log
```
